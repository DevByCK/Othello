# 오델로 게임 기획서 (GDD)

## 1. 목적과 범위
- 웹 브라우저에서 실행되는 2인 오델로 게임을 구현한다.
- 1차 릴리스 범위는 **인간 대 인간(Local 2P)** 모드로 제한한다.
- AI 대전, 온라인 멀티플레이, 리플레이 기능은 후속 확장 항목으로 정의한다.

## 2. 플랫폼 및 기본 요구사항
- 대상 플랫폼: 데스크톱/모바일 브라우저 (최신 Chrome, Safari, Edge)
- 보드 크기: 8x8 고정
- 시작 배치: 중앙 4칸에 흑/백 2개씩 표준 오델로 배치
- 플레이 시간: 턴 제한 없음

## 3. 핵심 게임 규칙
### 3.1 턴 진행
- 흑(선공)부터 시작한다.
- 자신의 돌을 놓을 수 있는 칸(유효 수) 중 한 칸에만 배치할 수 있다.
- 돌을 놓으면 가로/세로/대각선 방향으로 사이에 낀 상대 돌을 모두 뒤집는다.

### 3.2 유효 수 판정
- 빈 칸이어야 한다.
- 최소 한 방향 이상에서 `내 돌 - 상대 돌(1개 이상) - 놓을 칸` 또는 그 역순의 포위 형태가 성립해야 한다.
- 유효 수가 아닌 칸은 클릭해도 배치되지 않으며 안내 메시지를 표시한다.

### 3.3 패스 및 종료
- 현재 플레이어가 둘 수 있는 칸이 없으면 자동 패스한다.
- 양 플레이어 모두 연속으로 둘 수 없으면 게임 종료한다.
- 또는 보드 64칸이 모두 채워지면 즉시 종료한다.

### 3.4 승패 판정
- 종료 시 돌 개수(흑/백)를 계산한다.
- 더 많은 돌을 가진 플레이어가 승리한다.
- 개수가 같으면 무승부로 처리한다.

## 4. UX/UI 요구사항
### 4.1 화면 구성
- 게임 보드(8x8)
- 현재 턴 표시(흑/백)
- 돌 개수 표시(흑 n개 / 백 n개)
- 게임 상태 메시지(진행 중, 패스, 종료, 승/패/무)
- `새 게임` 버튼

### 4.2 상호작용
- 유효 수 위치는 시각적으로 하이라이트한다.
- 유효하지 않은 칸 클릭 시 사용자에게 이유를 짧게 안내한다.
- 턴 종료 후 상태 영역을 즉시 갱신한다.
- 게임 종료 시 추가 배치를 막고 결과 모달 또는 결과 영역을 표시한다.

## 5. 상태/데이터 스펙
- 보드 상태: 8x8 2차원 배열 (`empty`, `black`, `white`)
- 현재 턴: `black | white`
- 게임 상태: `playing | finished`
- 패스 여부 추적: 연속 패스 카운트(0~2)
- 점수: 흑/백 돌 개수 실시간 계산 또는 턴 종료 시 계산

## 6. 예외 및 에러 처리
- 잘못된 입력(보드 외 영역 클릭, 유효하지 않은 수)은 무시하고 상태 유지
- 진행 중이 아닐 때(종료 후) 보드 입력 차단
- 상태 불일치 감지 시(디버그 모드) 콘솔 경고 출력

## 7. 품질 기준(완료 정의)
- 모든 규칙(배치, 뒤집기, 패스, 종료, 승패)이 오델로 표준 규칙과 일치
- 최소 10개 이상의 규칙 테스트 케이스 통과
  - 단일 방향 뒤집기
  - 다중 방향 동시 뒤집기
  - 패스 발생/해제
  - 종료 조건 2종
  - 무승부 판정
- UI에서 턴/점수/결과가 규칙 엔진 상태와 항상 일치

## 8. 확장 항목(후속)
- AI 플레이어(난이도: Easy/Normal/Hard)
- 착수 히스토리 및 되돌리기(Undo)
- 사운드/애니메이션 강화
- 온라인 대전(방 생성/입장)
